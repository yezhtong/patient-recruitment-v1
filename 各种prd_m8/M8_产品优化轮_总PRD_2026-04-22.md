### M8.2 · 表单管理器 + AI 预筛（预计 1.5 周）

**范围**：
- 新 Prisma 模型：`TrialPrescreenForm` + `TrialPrescreenFormItem`（字段类型、必填、默认值、跳题规则）
- 后台**表单管理器**（可视化增删字段，拖拽排序，题型：单选/多选/文本/数字/日期/同意项）

- **AI 预筛生成**：运营贴广告原文 → AI 抽字段生成初版表单 → 运营二次编辑
- 患者端预筛页改造：从硬编码 JSON 切换到从 DB 读动态表单 schema
- LLM 调用日志 `LlmCallLog`（prompt、response、tokens、cost），管理端可查

**验收**：3 条真实试验都能通过后台管理器完整编辑其预筛表单；AI 生成字段准确率 ≥ 80%（人工抽检）

### M8.3 · 账号体系 + 社区重做（预计 2 – 2.5 周）

**范围**：

- User 模型扩展：`symptoms` / `aiDiseaseTags` / `medicalRecords` / `lastMatchedAt`

- 审核队列 UI 改造（新增"AI 判断"列 + 置信度 + 人工覆盖理由字段）

**验收**：tsc 零错、build 通过、`disease-matcher` 抽检 5 例 ≥ 4 例合理（决策 7 反转替代原 ICD-10 条数门槛）、匹配助手推荐 RT < 5s、`disease-matcher` RT < 5s、社区 AI 审核误杀率抽检 < 10%

## 四、共用技术基础

### 4.1 DeepSeek 接入抽象
- `src/lib/llm.ts`：`LlmProvider` 接口 + `DeepSeekProvider` 实现 + `MockLlmProvider` 开发回退
- 环境变量 `DEEPSEEK_API_KEY` / `DEEPSEEK_BASE_URL`（未配置自动 Mock，与 SMS 同模式）
- 统一调用日志：所有 LLM 调用写 `LlmCallLog`（含 prompt hash 以便去重）
- 错误兜底：LLM 超时/失败时降级为"暂不可用，稍后再试"

### 4.2 疾病匹配服务（决策 7 反转 · 2026-04-22）

> **废止** 原"ICD-10/11 词典 + 本地库"方案，改用 DeepSeek AI 动态分析。

- 核心 lib：`src/lib/disease-matcher.ts`
  - `analyzeSymptoms(freeText): DiseaseTag[]` —— 输入自由文本，调 DeepSeek 输出 `[{label, keyword, confidence}]`
  - `recommendGroups(userTags, groups, mode)` —— 基于用户 `aiDiseaseTags` 与分区 `recommendedDiseaseKeywords` 做语义匹配（mode=`string`|`semantic`）
- **无需**外部数据：不跑 `seed-icd10.ts`、不建 `Icd10Term` 模型、不维护 2 万条 ICD 条目
- 调用日志：所有 DeepSeek 调用写 `LlmCallLog`（scenario=`disease_analysis`）
- 缓存：prompt hash 60s 内命中返回上次结果
- 兜底：`DEEPSEEK_API_KEY` 未配置时走 Mock，返回 3 条 fixture 标签（沿用 M8.2 `LlmProvider` 模式）

### 4.3 行为日志与限流（M8.1 落地）
- 新 Prisma 模型：`UserBehaviorLog`（userId/guestToken/action/targetType/targetId/ua/ip/createdAt）
- 新 Prisma 模型：`AccountLock`（userId/reason/lockedAt/unlockedAt/appealText/appealStatus/handlerId）
- 埋点：试验详情页、预筛页访问时 server-side 写一条
- 限流检查：`src/lib/rate-limit.ts`（查最近 30min 的 `trial_detail_view` 计数）
- 游客追踪：首次访问写 httpOnly cookie `jt_guest_token`（UUID）

### 4.4 图片素材管理
- `public/images/` 目录结构：`hero/` `community/` `trials/` `avatar/`
- 后台页 `/admin/media`（M8.1 搭骨架）：运营上传 → 自动压缩 + 生成 URL → 在试验/社区引用
- 占位策略：无素材时走默认色块 + 文字，不暴露"图片丢失"
