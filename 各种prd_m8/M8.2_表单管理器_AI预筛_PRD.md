- 作为患者，遇到表单加载失败（网络问题）时，我能看到清楚的错误提示和"重新加载"按钮，而不是白屏

- M8.2 不涉及 AI 账号（系统账号要等到 M8.3）

### admin / operator
- 作为运营，当我拿到一条新试验的招募广告（.doc/.pdf/文本），我可以**贴进后台一个框**，点"AI 生成"，**30 秒内**得到一份初版预筛表单（字段名 + 题型 + 必填/选填 + 选项），我二次编辑即可发布

## 四、数据模型增量

```prisma
// M8.2 · 试验预筛表单（一条 ClinicalTrial 对应 0..1 条 Form）
model TrialPrescreenForm {
  id            String   @id @default(cuid())
  trialId       String   @unique
  trial         ClinicalTrial @relation(fields: [trialId], references: [id], onDelete: Cascade)
  version       Int      @default(1)         // 每次大改 +1，便于追溯
  isPublished   Boolean  @default(false)     // 未发布时 /prescreen 降级到通用版
  title         String?                       // 可选"xxx 试验预筛问卷"
  description   String?                       // 可选：顶部说明
  successMessage String?                      // 提交后的成功文案（覆盖默认）

  items         TrialPrescreenFormItem[]
  generatedBy   String?                       // "ai" / "human" / "ai-then-edited"
  generatedAt   DateTime?
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  @@index([isPublished])
}

// 字段定义
model TrialPrescreenFormItem {
  id            String   @id @default(cuid())
  formId        String
  form          TrialPrescreenForm @relation(fields: [formId], references: [id], onDelete: Cascade)

  // 字段定义
  fieldKey      String                        // 存入 Lead.projectAnswers 的 key（英文 snake_case）
  label         String                        // 问题文本（中文）
  helpText      String?                       // 辅助说明
  fieldType     String                        // single / multi / text / textarea / number / date / agree
  options       String?                       // JSON 字符串：[{value, label}, ...]
  placeholder   String?
  defaultValue  String?                       // JSON 字符串
  isRequired    Boolean @default(false)
  minValue      Float?                        // 数字/文本长度 min
  maxValue      Float?                        // 数字/文本长度 max
  regex         String?                       // 文本自定义正则
  errorMessage  String?                       // 校验失败文案

  // 跳题规则（仅前端读）
  showWhen      String?                       // JSON: {fieldKey, op, value} op=eq|neq|in|notIn

  sortOrder     Int      @default(100)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  @@index([formId, sortOrder])
}

// LLM 调用日志
model LlmCallLog {
  id            String   @id @default(cuid())
  operatorId    String?                       // 触发者（operator）
  scenario      String                        // prescreen_generate / match_assistant / community_review ...
  provider      String                        // deepseek / mock
  model         String?                       // deepseek-chat / mock
  promptHash    String                        // 用于去重/查重
  promptChars   Int
  responseChars Int?
  promptTokens  Int?
  completionTokens Int?
  totalTokens   Int?
  estimatedCostCny Float?                     // 精度到 0.0001
  durationMs    Int?
  status        String                        // success / error / timeout
  errorMessage  String?
  createdAt     DateTime @default(now())

  @@index([scenario, createdAt])
  @@index([provider, status, createdAt])
}
```

## 五、业务规则详解

### 5.1 LlmProvider 抽象

- 文件：`src/lib/llm.ts`
- 接口：
  ```ts
  export interface LlmProvider {
    chat(input: { system?: string; user: string; maxTokens?: number; temperature?: number }):
      Promise<{ text: string; usage?: { prompt: number; completion: number; total: number } }>;
  }
  ```
- 实现：
  - `DeepSeekProvider`：调 DeepSeek Chat API，`model="deepseek-chat"`，timeout 30s
  - `MockLlmProvider`：开发期回退，按 scenario 返回硬编码 fixture
- 工厂：`getLlmProvider()` 读 `DEEPSEEK_API_KEY` 是否存在决定返回哪个（与 `getSmsProvider()` 同模式）
- **所有**调用必须通过 `callWithLogging(scenario, fn)` 包装，自动写 `LlmCallLog`

### 5.2 表单管理器交互规则

- **左侧字段列表**（拖拽排序）：
  - 每个字段条目显示：题型图标 + label + [必填] 徽章 + 排序手柄
  - 点击条目高亮 + 右侧展开编辑面板
  - 拖拽变更 `sortOrder`（整数 step=10，便于后续插入）
- **右侧编辑面板**字段：
  - 题型 dropdown（single / multi / text / textarea / number / date / agree）
  - fieldKey（英文，snake_case，实时校验唯一性）
  - label（中文）
  - helpText（可选）
  - 选项配置（仅 single / multi 题型显示，value + label 成对）
  - 必填开关
  - 默认值
  - 最小值/最大值（number/text length）
  - 自定义正则 + 错误提示
  - 跳题规则（可视化：`当 <fieldKey下拉> <op下拉> <值> 时 显示/隐藏此题`）
- **顶部工具栏**：
  - 「+ 新增字段」按钮
  - 「预览」按钮 → 弹窗/侧抽屉展示患者端渲染
  - 「AI 生成初版」按钮 → 弹窗贴广告原文
  - 「保存草稿」（不影响已发布版本）
  - 「发布」（将当前 version 变为 isPublished=true，version+1）
  - 「版本历史」（M8.2 暂不做，v2 补）

### 5.3 AI 生成预筛表单

- 运营点「AI 生成初版」：
  1. 弹窗提示：「贴入招募广告全文（支持纯文本，≤ 4000 字）」
  2. 文本框 + 字数计数
  3. 点「开始生成」→ loading（"AI 正在读广告并抽取预筛问题，约需 20-30 秒"）
  4. 调 `/api/admin/trials/[id]/generate-form`
- 后端：
  - Prompt（system 部分）大意：
    > 你是临床试验预筛表单抽取专家。我会给你一条试验的招募广告。请抽取出**筛选患者**所需要的预筛问题，输出为 JSON 数组，每项包含：fieldKey（英文 snake_case）/ label（中文问题）/ fieldType（single/multi/text/number/date）/ options（如果是单选/多选）/ isRequired（默认 true）/ helpText（可选，补充说明）。
    > 注意：只抽取**与入排标准直接相关**的问题（年龄、性别、诊断、病程、既往治疗、过敏史等），忽略联系方式和知情同意（这些我会统一加）。
    > 输出格式示例：[{"fieldKey":"...","label":"...","fieldType":"...","options":[...],"isRequired":true}]
  - user 部分：广告原文
  - 返回 JSON → 解析 → 写 `TrialPrescreenForm` + items（isPublished=false，generatedBy="ai"）
  - 同时写 `LlmCallLog`（scenario=prescreen_generate）
- 前端：生成成功后刷新表单管理器，所有字段可编辑
- **准确率抽检口径（≥ 80%）**：
  - 用 M5 seed 的 3 条真实试验（乳腺癌 / 高血压 / 额头纹）分别跑一次
  - 人工对照 `真实招募项目/parsed-trials.md` 里的入排标准，算每条的**字段召回率**（真标准命中的字段 / 总标准字段）
  - 3 条平均召回率 ≥ 80% 通过
  - **不看字段名翻译质量**（label 接近即可，fieldKey 可人工改）

### 5.4 患者端预筛页改造

- `/prescreen/[slug]`：
  1. server 查 `ClinicalTrial` by slug
  2. 查 `TrialPrescreenForm where trialId=... and isPublished=true` 取最新 version
  3. 有 → 渲染动态表单（按 items sortOrder）；无 → 渲染 M1 通用预筛（兜底）
- 动态渲染：
  - 每个字段按 fieldType 映射组件（`<Radio />` / `<Checkbox />` / `<Input />` / `<Textarea />` / `<NumberInput />` / `<DatePicker />` / `<Agree />`）
  - 跳题规则客户端实时求值（watch react-hook-form 的 getValues）
  - 提交时只提交"当前可见"字段（隐藏字段视为未填）
- **提交逻辑**：与 M1 一致，写入 `Lead.projectAnswers`（JSON 字符串，key=fieldKey）

### 5.5 LLM 调用日志后台 `/admin/llm-logs`

- 列表：
  - 时间 / scenario / provider / model / promptChars / totalTokens / cost / 状态 / 操作（查看详情）
- 详情（抽屉）：
  - 完整 prompt（折叠展开）
  - 完整 response（折叠展开）
  - 元信息：耗时、错误（如有）
- 顶部统计：
  - 今日调用次数 / 今日花费
  - 近 7 天成功率
  - 按 scenario 分布（M8.2 只有 prescreen_generate，M8.3 会补）
- **脱敏**：prompt/response 里如含手机号/身份证号，管理端展示前在 server 侧打星号（M8.2 基础版本做简单正则替换）

### 5.6 图片素材库自动压缩（M8.1 遗留 → M8.2 补）

- 上传 pipeline：
  - 原图 > 500KB 且为 JPEG/PNG/WebP → 用 `sharp` 压缩到 ≤ 500KB，同时生成 webp 版本
  - 存盘命名：`{cuid}.{ext}` + `{cuid}.webp`
  - `MediaAsset.url` 指向 webp（若浏览器不支持 fallback 到原格式，用 `<picture>` 标签）
- 图片被引用时优先返回 webp

## 六、异常流

| 场景 | 行为 |
|---|---|
| DEEPSEEK_API_KEY 未配置 | `getLlmProvider()` 返回 `MockLlmProvider`，后台 AI 生成返回 fixture（3 条固定示例字段） + 日志标记 provider=mock |
| LLM 调用超时（> 30s） | 写日志 status=timeout，前端弹红色 toast："AI 生成超时，请稍后重试。你可以手动创建字段。" |
| LLM 返回非 JSON | 尝试正则抽取 JSON 块；失败则 status=error，前端提示"AI 响应格式异常，请重试" |
| LLM 调用失败（API error） | 写日志 status=error + errorMessage；前端提示，保留已有字段不覆盖 |
| 运营点生成但广告文本 > 4000 字 | 前端拦截，提示"请精简到 4000 字以内或分段生成" |
| 保存字段时 fieldKey 重复 | 服务端校验 422 + 提示"字段 key 不能重复" |
| fieldKey 非 snake_case | 服务端校验 422 + 提示"字段 key 只能使用小写字母、数字和下划线" |
| 患者端加载表单失败（网络） | 显示错误态 + "重新加载"按钮（不走降级，避免用户意外走到通用表单） |
| 运营删除一个已引用的字段 | 校验：如果已有 Lead 在 `projectAnswers` 里使用该 fieldKey → 提示"该字段已有 N 条线索引用，确认删除？"，删除后 Lead 历史数据保留 |
| 表单未发布时患者访问 | 降级到 M1 通用预筛（之前的硬编码版），不报错 |
| 跳题规则引用了不存在的 fieldKey | 保存前校验 422 + 提示哪条规则有问题 |

## 八、验收标准

### 8.1 用户能...

- 运营能在后台一键生成 3 条真实试验的预筛表单初版
- 运营能拖拽排序字段、增删改字段、设置跳题
- 运营能预览患者端渲染效果
- 运营能发布 / 回滚版本
- 患者访问 `/prescreen/[slug]` 能看到该试验的定制表单（已发布）
- 患者按跳题规则只看到相关问题
- 运营能在 `/admin/llm-logs` 看到所有 LLM 调用

### 8.2 用户看到...

- AI 生成弹窗里有"AI 结果需人工复核"的提示
- LLM 调用失败时有明确错误信息（不是白屏）
- 表单未发布时患者看到"通用预筛"（降级）

### 8.3 用户不能...

- 运营不能发布 `fieldKey` 重复的表单
- 运营不能用非法格式的 fieldKey
- 运营不能配置引用不存在字段的跳题规则
- 患者不能提交被跳题规则隐藏的字段（服务端再校验一次）

### 8.4 技术硬门槛

- `npx tsc --noEmit` 零错
- `npm run build` 通过
- 3 条真实试验都能通过管理器完整编辑其预筛表单
- AI 生成字段**平均召回率 ≥ 80%**（抽检口径见 5.3）
- LLM 所有调用必须有 `LlmCallLog` 记录（在日志里能查到）
- 未配置 DEEPSEEK_API_KEY 时自动走 Mock，页面仍可工作（类似 SMS dev 模式）
