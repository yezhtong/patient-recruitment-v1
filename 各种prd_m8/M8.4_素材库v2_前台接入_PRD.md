## 0. 为什么做这个

今天用户上传一张 hero 图后，发现首页**完全没变化**——因为前台从未读 `MediaAsset` 表，首页 hero、3 步配图、故事头像、社区头图、试验顶图全部硬编码 Unsplash URL。

M8.1 PRD 原文写过"PRD 不强依赖素材 · 搭好引用框架即可"，当时只做了素材库的"后台管理侧"，**前台消费侧从未接通**。

同时用户希望 hero 区不再是单张静态图，改为**轮播（3-5 张）**，且每张图左下角的 overlay 文案（研究中心标签 + 医院门诊描述）要**跟图一起切换**。

这三件事合在一起，就是 M8.4。

**核心要求**：所有联动配置放后台，运营不写代码就能换图、改文案、调顺序。

## 二、首页 Hero 轮播规格

### 2.1 张数与切换

- **张数**：默认目标 3-5 张；实际以后台启用的 `hero` 分类图片数为准
- **切换方式**：**自动 + 手动都支持**
  - 自动：默认间隔 **6 秒**（中老年用户阅读 overlay 文案 + 看图需要 ≥5s，6s 是阅读节奏的合理下限）
  - 手动：图下方提供圆点指示器（dot indicator），点击切到对应张；鼠标悬停/键盘焦点进入时**自动暂停**
- **过渡效果**：淡入淡出（fade，时长 400ms）。不用横向滑动——避免页面横向位移影响老人眼睛
- **首张加载**：`loading="eager"` + `fetchpriority="high"`，其余懒加载

### 2.2 Overlay 文案跟图切

每张轮播图左下角都有一条 overlay（现有样式 `.hero-photo__overlay` + `.hero-photo__overlay-label` + `.hero-photo__overlay-text`），结构为：

```
[标签：真实研究中心]
[正文：北京协和医院 · 乳腺肿瘤门诊]
```

切图时 overlay 文案**跟着切**，不允许出现"图切了文案没切"或"图没切文案切了"的错位。

### 2.3 Overlay 文案存储方案 —— 推荐 **方案 B**

| 方案 | 做法 | 优点 | 缺点 | 工作量 | pm 评估 |
|---|---|---|---|---|---|
| A | 复用 `MediaAsset.note`，约定 `"标签\|正文"` 格式 | 零 schema 变更，最快 | 运营要记约定格式，容易写错；老数据的 note 可能已有其他含义；前端要 split 解析 | 0.5d | 不推荐 |
| **B** | **扩 `MediaAsset` 加 `overlayLabel` + `overlayText` 两字段** | **字段语义清晰；Admin 表单直观；老数据字段为空不影响** | 动 schema + 一次 migrate | 1d | **推荐** |
| C | 新建 `MediaHeroSlide` 关联表 | 最规整，支持 slide 独立排序、独立启停 | 工作量翻倍；引入新模型增加心智；首期过度设计 | 2-3d | V2 再做 |

**pm 推荐方案 B 的理由**：
1. M8.4 只动 hero 一个接入位需要 overlay，另外 5 个接入位都不需要——不值得新建表
2. `MediaAsset` 表里两个可空字段，非 hero 分类的素材留空即可，不污染数据
3. 排序问题用 `createdAt desc` 兜底，不足以为此新建表；若 V2 有复杂排序需求再升级到方案 C

### 2.4 空态分层

| hero 分类启用的图数 | 表现 |
|---|---|
| 0 | 走 Unsplash 兜底图（保留现有 `heroImageSrc` 常量）+ 硬编码 overlay 文案"真实研究中心 / 北京协和医院 · 乳腺肿瘤门诊" |
| 1 | 单图，不轮播；overlay 取该图的 `overlayLabel/overlayText`，若都空则走兜底 overlay 文案 |
| 2-5 | 进入轮播模式；每张 overlay 各自渲染；overlay 字段空的图 overlay 区不显示（不显示空白块） |
| ≥6 | 只取最新 5 张（按 `createdAt desc`）——避免老图堆积让 hero 转不完 |

### 2.5 可访问性（a11y）

- 轮播容器 `role="region"` + `aria-roledescription="carousel"` + `aria-label="首页主视觉"`
- 每张 slide 是 `role="group"` + `aria-roledescription="slide"` + `aria-label="第 N 张，共 M 张"`
- **必须有暂停按钮**（右下角小图标，中老年友好尺寸 ≥ 44×44）
- 鼠标悬停 / 键盘 focus 进入轮播区自动暂停
- 遵循 `prefers-reduced-motion: reduce`——开启此偏好的用户直接展示第一张，不自动切换
- Dot 指示器按钮 `aria-label="切到第 N 张"`，当前张 `aria-current="true"`

## 三、后台联动配置规格

### 3.1 运营能做什么（M8.4 必做）

**在现有 `/admin/media` 页面上增强**，不新建独立页（理由见 3.2）：

1. **编辑每张图的 overlay 文案**（仅 hero 分类显示此字段）
   - 新增"编辑"按钮（现有只有"复制 URL / 删除"两个操作）
   - 编辑弹窗含：标签（≤10 字）、正文（≤30 字）、备注 note、启用/停用开关
2. **启停开关显式化**
   - 当前 `isEnabled` 只有软删能设 false，没有"暂停展示但保留文件"的操作
   - 新增独立的"暂停展示 / 恢复展示"切换（区别于"删除"）
3. **调整显示顺序（hero 分类）**
   - 上移 / 下移按钮（影响 `createdAt` 或新增 `sortOrder` 字段——建议**不新增字段**，用"重写 createdAt 为 now()"近似上移，简化实现。**这点列入待拍板**）

### 3.2 为什么不新建 `/admin/hero-slides` 独立页

1. 运营心智成本：一个"图片库"管所有图比两个入口清晰
2. hero 轮播本质还是"一组 hero 分类的图"，和其他分类结构一致，分开反而割裂
3. 编辑表单通过"仅在 hero 分类显示 overlay 字段"做条件渲染即可

### 3.3 运营不做的事（避免过度设计）

以下功能**不做**，写入 future-backlog：
- ✗ 指定"这张图专用在哪个试验/分区"——接入位 4、5 延后到 M8.5
- ✗ 排期（定时启用/停用）
- ✗ A/B 多版本 hero 组
- ✗ 裁剪 / 滤镜 / 水印

## 四、数据模型变更（方案 B 增量）

```prisma
model MediaAsset {
  // ...现有字段保持不变
  overlayLabel String?   // 新增：hero 分类专用，左下角标签（如"真实研究中心"）
  overlayText  String?   // 新增：hero 分类专用，左下角正文（如"北京协和医院 · 乳腺肿瘤门诊"）
  // ...
}
```

**Migration 名**：`m84_media_overlay_fields`

**数据兼容**：
- 两字段均可空，老数据自动为 NULL，不影响现有逻辑
- 非 hero 分类的图写入这两个字段也不会报错（但前端不会渲染）

**不改字段**：
- 不加 `sortOrder`（详见 3.1 与第九章待拍板）
- 不加 `usage`/`targetId`（接入位 4、5 的指定使用场景延后）

## 五、分类语义定义

| 分类键 | 一句话定义 | 用法示例 | 是否新增 |
|---|---|---|---|
| `hero` | 首页主视觉大图（轮播） | 研究中心实景、医患场景、真实门诊 | 已有 |
| **`step`** | **首页"接下来会发生什么"3 步流程配图** | **填资料场景 / 电话联系 / 门诊评估** | **新增** |
| `avatar` | 患者故事头像（马赛克化/无法识别个人） | 首页 3 张故事头像、社区未上传头像时的占位 | 已有 |
| `community` | 社区分区的头图（接入位 4，M8.5 落地） | 乳腺癌分区、糖尿病分区的顶图 | 已有（本期不接入前台） |
| `trial` | 试验详情页顶图（接入位 5，M8.5 落地） | 研究中心环境图、病种相关插画 | 已有（本期不接入前台） |
| `faq` | FAQ 页配图 | V1 无独立 FAQ 页 | 已有（本期不接入前台） |

**关于新增 `step` 分类**：
- 理由：首页 3 步流程是独立场景，语义和 hero 不同（hero 是主视觉，step 是流程示意），放在同一个 hero 分类会混淆
- 运营界面：在 `/admin/media` 的分类下拉中多一个"首页流程 3 步"
- 数量约束：前端严格取最新 3 张，超出忽略，不足走 Unsplash 兜底

## 六、降级与空态

每个接入位都要保证"素材不存在也不白屏、不图崩"：

| 接入位 | 0 张时表现 | 加载失败（onError）表现 |
|---|---|---|
| 首页 hero | Unsplash 兜底 + 硬编码 overlay | `onError` 切 Unsplash 兜底（前端 img 标签加 onError handler） |
| 首页 step（3 步） | 3 张 Unsplash 兜底（即当前 page.tsx 硬编码的那 3 张） | 单张失败切对应兜底 URL |
| 首页故事头像（3 张） | 走灰底 + 首字字符头像（用 `.avatar.avatar--lg` 现有 CSS） | 同 0 张 |
| 社区分区头图 | 分区色块（每分区有色号）+ 文字标题 | 同 0 张 |
| 试验详情顶图 | 乳白色块 + 试验标题 + 病种 tag（现有 trial-card 顶部风格复用） | 同 0 张 |

**技术提示（给工程师，不是规格）**：`<img>` 的 `onError` 在 Server Component 里不生效，需要 Client Component 包壳或用 CSS `background-image` + 预检方案。具体方案由 tech-lead 定。

## 七、验收标准（产品视角粗颗粒）

**用户能看到 / 能做 / 不能怎样**：

1. ✓ 运营在后台上传 3 张 hero 图 + 填写各自 overlay 文案后，**首页刷新能看到轮播**，文案跟图切换，无错位
2. ✓ 运营上传 1 张 hero 图时，首页展示**单图不轮播**，overlay 取该图文案（或兜底）
3. ✓ 运营全部停用 hero 图时，首页展示 Unsplash 兜底图 + 硬编码文案，**不白屏、不 404**
4. ✓ 用户鼠标悬停或键盘 Tab 进入 hero 区，**轮播自动暂停**；移开后恢复
5. ✓ 用户点击 dot 指示器能切到指定张；点击暂停按钮能停止自动切
6. ✓ 用户开启系统"减少动画"偏好时，轮播自动停在第一张，不切换
7. ✓ 运营在 `/admin/media` 的 hero 分类能看到 overlay 文案编辑入口，非 hero 分类看不到
8. ✓ 首页 3 步配图、3 张故事头像同样接入素材库，且"0 张时走兜底"行为正确
9. ✓ `pnpm tsc --noEmit` 通过；`pnpm build` 通过
10. ✓ hero 区首屏 LCP 不劣化超过 200ms（新增轮播 JS 不应拖慢首屏，需要测一次）

## 十、风险与护栏

- **医疗合规**：运营上传图/写 overlay 文案时，禁止出现疗效对比、治愈承诺。`/admin/media` 上传表单底部加一行小字提示："图片需为授权/原创素材；文案不得含疗效承诺或治愈暗示（M8.5 考虑接敏感词库）"
- **素材原始性存疑**：V1 运营上传图片完全凭自觉，无版权校验——列入 future-backlog 的"素材合规审核流"
- **首屏性能**：轮播 JS 是首屏关键路径，T4 交付时必须 Lighthouse 跑一次 LCP 对比
- **中老年可用性**：轮播默认速度 6s 是假设，上线 2 周内必须看埋点（用户是否真的看到第 2、3 张）——指标"hero_slide_view_depth"记入 M8.5 埋点清单
