适用场景：

## 设计原则

### 1. 患者语言优先

避免使用登记平台式术语堆叠。页面文案优先回答患者最关心的 4 个问题：

1. 这项试验是做什么的
2. 我大概率适不适合
3. 参加后大概会经历什么
4. 我下一步怎么做

### 2. 一个页面一个主目标

每个页面尽量只有一个主 CTA：

1. 首页：开始找试验
2. 试验详情页：立即预筛
3. 社区帖子详情页：查看相关试验 / 立即预筛
4. 预筛页：提交预筛

### 3. 统一组件优先

为了适配短周期实现，公开站尽量复用统一组件：

1. `TrialCard`
2. `FAQCard`
3. `CommunityPostCard`
4. `SectionHeader`
5. `PrimaryCTA`
6. `SecondaryCTA`
7. `EmptyState`
8. `LoginPrompt`

### 4. 账户轻量化

患者账户只服务于：

1. 登录/注册
2. 基本资料复用
3. 查看我的报名
4. 参与社区

### 5. 社区辅助主链路

社区是强需求，但角色是“信任和导流模块”。社区页面必须始终能导向：

1. 相关试验详情页
2. 立即预筛入口

## 6. 典型病种 / 适应症建议

为了方便首页入口、筛选器和社区分区设计，V1 可优先支持以下示例：

### 传统病种

1. 肺癌
2. 乳腺癌
3. 银屑病
4. 特应性皮炎
5. 罕见病

### 医美相关适应症

1. 痤疮
2. 黄褐斑
3. 瘢痕修复
4. 脱发 / 雄激素性脱发
5. 色素沉着相关适应症
6. 皮肤年轻化 / 光电治疗相关适应症

## 全局信息与组件规范

## 1. 全局导航

### 顶部导航

1. 首页
2. 找试验
3. 患者社区
4. 患者帮助
5. 联系我们
6. 登录 / 我的

### 全局 CTA 文案

| 场景 | 主按钮文案 | 备选文案 |
|---|---|---|
| 找项目 | 开始找试验 | 查看招募项目 |
| 试验转化 | 立即预筛 | 先做资格判断 |
| 社区导流 | 查看相关试验 | 我想了解这个项目 |
| 登录 | 手机号登录 | 登录后继续 |
| 发帖 | 发布帖子 | 提交问题 |

## 2. 统一卡片字段

### `TrialCard`

| 字段 | 是否必需 | 说明 |
|---|---|---|
| 试验标题 | 是 | 患者友好标题，不直接使用过长官方标题 |
| 病种标签 | 是 | 如肺癌、乳腺癌、痤疮、黄褐斑、瘢痕修复 |
| 城市/医院 | 是 | 至少显示城市，优先显示中心医院 |
| 招募状态 | 是 | 招募中 / 即将开始 / 暂停招募 |
| 一句话说明 | 是 | 15 到 30 字概括 |
| CTA | 是 | 查看详情 |

### `CommunityPostCard`

| 字段 | 是否必需 | 说明 |
|---|---|---|
| 帖子标题 | 是 | 50 字以内最佳 |
| 帖子类型 | 是 | 提问 / 经验分享 |
| 病种标签 | 是 | 按病种分区 |
| 作者展示名 | 是 | 昵称或匿名用户 |
| 发布时间 | 是 | 相对时间或日期 |
| 评论数 | 否 | 可显示 |
| 关联试验标签 | 否 | 有关联项目时展示 |

### `FAQCard`

| 字段 | 是否必需 | 说明 |
|---|---|---|
| 标题 | 是 | 问题式标题 |
| 摘要 | 是 | 40 到 80 字摘要 |
| 分类 | 是 | 临床试验说明 / 报名流程 / 风险说明 |
| CTA | 是 | 查看详情 |

## 3. 统一状态文案

### 试验状态

1. 招募中
2. 即将开始
3. 已结束

### 我的报名状态

1. 已提交
2. 待联系
3. 跟进中
4. 当前不符合
5. 已关闭

### 社区帖子状态

1. 已发布
2. 审核中
3. 未通过

## 页面清单

## P01 首页

### 页面目标

1. 建立基础信任
2. 引导进入找试验
3. 让患者知道可以通过 FAQ 和社区进一步了解

### 主要模块

| 模块 | 核心字段 | 说明 |
|---|---|---|
| 顶部导航 | Logo、导航、登录入口 | 全站统一 |
| Hero 区 | 主标题、副标题、主 CTA、次 CTA | 第一屏完成价值表达 |
| 重点病种入口 | 病种 / 适应症名称、图标/标签、入口按钮 | 3 到 8 个重点病种 / 适应症 |
| 重点试验推荐 | `TrialCard` 列表 | 4 到 8 个项目 |
| 平台流程说明 | 步骤标题、步骤说明 | 解释找试验到提交的流程 |
| FAQ 入口 | `FAQCard` 列表 | 2 到 4 条高频问题 |
| 社区入口 | 社区简介、精选帖子 | 导向患者社区 |
| 底部信任信息 | 免责声明、联系方式 | 页面收口 |

### 原型文案建议

#### Hero

- 主标题：`帮你找到合适的临床试验机会`
- 副标题：`按疾病和城市快速筛选项目，先做资格判断，再由招募顾问跟进。`
- 主按钮：`开始找试验`
- 次按钮：`先了解临床试验`

#### 平台流程说明

- 标题：`参与临床试验，大致会经历这 4 步`
- 步骤 1：`找到相关项目`
- 步骤 2：`完成资格预筛`
- 步骤 3：`招募顾问联系你`
- 步骤 4：`进入后续筛选流程`

#### 社区入口

- 标题：`先看看病友怎么说`
- 描述：`如果你还在犹豫，可以先看看同病种或同类适应症用户的提问和经验分享。`
- 按钮：`进入患者社区`

### 实现备注

1. 首页首屏只保留一个主 CTA：`开始找试验`
2. 重点试验和重点病种 / 适应症可以先后台手动配置
3. 社区和 FAQ 只展示少量精选内容，不做复杂推荐

## P02 试验列表页

### 页面目标

1. 让患者按条件快速找到相关试验
2. 让患者继续进入试验详情页

### 主要模块

| 模块 | 核心字段 | 说明 |
|---|---|---|
| 搜索筛选区 | 病种、城市、招募状态 | V1 保持简单 |
| 结果统计 | 当前结果数 | 如“共 18 个项目” |
| 试验列表 | `TrialCard` 列表 | 主体内容 |
| 空状态 | 空结果文案、返回 CTA | 提升留存 |
| 侧边帮助区 | FAQ 入口、联系顾问入口 | 非必须，可放下方 |

### 字段建议

| 字段 | 类型 | 说明 |
|---|---|---|
| 病种 | 下拉/标签 | 必选筛选字段 |
| 城市 | 下拉/标签 | 必选筛选字段 |
| 招募状态 | 单选 | 默认只显示招募中 |

### 原型文案建议

- 页面标题：`查找适合你的临床试验`
- 筛选区提示：`你可以先按疾病和城市缩小范围`
- 空状态标题：`暂时没有找到完全匹配的项目`
- 空状态说明：`你可以换一个城市试试，或者先看看相关病种 / 适应症的常见问题和用户讨论。`
- 空状态按钮 1：`重新筛选`
- 空状态按钮 2：`进入患者社区`

### 实现备注

1. V1 不做全文检索，优先做结构化筛选
2. 列表排序默认按后台配置顺序或最近更新时间

## P03 试验详情页

### 页面目标

1. 让患者理解这项试验
2. 让患者判断自己是否值得继续预筛
3. 把患者导向预筛或社区

### 主要模块

| 模块 | 核心字段 | 说明 |
|---|---|---|
| 顶部摘要区 | 项目标题、病种、城市、状态 | 第一眼信息 |
| 一句话说明 | 患者友好摘要 | 解释项目做什么 |
| 你可能适合吗 | 关键纳入条件、关键排除条件 | 只展示易懂版本 |
| 参与流程 | 预计步骤说明 | 降低未知感 |
| 研究中心信息 | 城市、医院、联系方式说明 | 可简化 |
| 主 CTA 区 | 立即预筛、联系顾问 | 核心转化 |
| 相关 FAQ | FAQ 列表 | 辅助解释 |
| 相关社区内容 | 帖子列表 | 辅助信任 |

### 字段建议

| 字段 | 是否必需 | 说明 |
|---|---|---|
| 患者友好标题 | 是 | 页面 H1 |
| 官方项目标题 | 否 | 可折叠展示 |
| 病种 | 是 | 标签 |
| 城市 | 是 | 标签 |
| 招募状态 | 是 | 状态 |
| 一句话说明 | 是 | 30 到 60 字 |
| 适合人群说明 | 是 | 文本块 |
| 不适合人群说明 | 是 | 文本块 |
| 流程说明 | 是 | 3 到 5 步 |
| 预筛按钮 | 是 | 主按钮 |

### 原型文案建议

- 标题示例 1：`针对晚期肺癌患者的靶向治疗研究`
- 一句话说明 1：`如果你正在寻找新的治疗机会，这项研究可能适合符合条件的肺癌患者。`
- 标题示例 2：`针对中重度痤疮人群的皮肤治疗研究`
- 一句话说明 2：`如果你长期受痤疮困扰，这项研究可能适合符合条件的相关人群。`
- 模块标题：`你可能适合这项研究吗？`
- 模块标题：`大概会经历什么？`
- 主按钮：`立即预筛`
- 次按钮：`先看看同类讨论`

### 实现备注

1. 页面必须同时提供 `立即预筛` 和 `社区入口`
2. 纳入排除条件需要翻译成患者能看懂的话
3. FAQ 和社区内容都应支持后台手动关联

## P04 登录 / 注册页

### 页面目标

1. 让患者快速建立账户
2. 承接预筛和社区的登录需求

### 主要模块

| 模块 | 核心字段 | 说明 |
|---|---|---|
| 标题区 | 页面标题、说明文案 | 解释为什么要登录 |

| 协议区 | 用户协议、隐私政策勾选 | 必要合规 |
| 登录成功回跳 | 来源页回跳标识 | 关键实现点 |

### 原型文案建议

- 标题：`登录后继续`
- 说明：`登录后你可以保存资料、提交预筛、查看报名记录，并参与社区讨论。`
- 手机号占位：`请输入手机号`
- 验证码占位：`请输入验证码`
- 按钮：`登录 / 注册`

### 实现备注

1. 一个页面同时承接登录和注册
2. 必须支持从预筛页、发帖页回跳

## P05 预筛表单页

### 页面目标

1. 收集该项目所需的资格信息
2. 让患者顺利提交

### 主要模块

| 模块 | 核心字段 | 说明 |
|---|---|---|
| 页面标题区 | 项目标题、进度提示 | 提示当前正在报名哪个项目 |
| 基础资料区 | 姓名、年龄、性别、手机号、城市 | 可从账户资料预填 |
| 项目特有问题区 | 项目级自定义字段 | 由后台配置 |
| 说明区 | 隐私说明、提交后流程 | 增加信任 |
| 提交区 | 提交按钮 | 主 CTA |

### 基础资料字段建议

| 字段 | 类型 | 必填 |
|---|---|---|
| 姓名 | 文本 | 是 |
| 性别 | 单选 | 是 |
| 年龄 | 数字 | 是 |
| 手机号 | 文本 | 是 |
| 所在城市 | 单选/文本 | 是 |
| 主要疾病 | 文本/单选 | 是 |

### 项目特有字段示例

| 字段 | 类型 | 示例文案 |
|---|---|---|
| 是否确诊某病种 | 是/否 | `是否已由医生确诊为非小细胞肺癌？` |
| 是否符合某适应症 | 是/否 | `是否存在中重度痤疮困扰并持续超过 6 个月？` |
| 当前治疗阶段 | 单选 | `你目前处于哪种治疗阶段？` |
| 是否做过基因检测 | 是/否 | `是否做过相关基因检测？` |
| 既往治疗情况 | 多选 | `你是否接受过以下治疗方式？` |

### 原型文案建议

- 标题：`开始做资格预筛`
- 说明：`这份问卷大约需要 2 到 4 分钟，提交后会由招募顾问进一步联系你。`
- 提交按钮：`提交预筛`
- 页尾说明：`提交信息仅用于项目匹配和后续联系。`

### 实现备注

1. 优先采用单页表单实现，减少多步骤复杂度
2. 已登录用户应自动预填基础资料
3. 项目字段只使用受控字段组件库

### 主要模块

| 模块 | 核心字段 | 说明 |

## 给设计和开发的最后约束

1. 试验详情页必须始终比社区页更接近主转化路径。
2. 社区页必须始终带有相关试验或预筛入口。
3. 账户页字段必须尽量与预筛基础资料字段复用。
4. FAQ、社区、试验详情三者之间必须能互相跳转。
5. 所有页面的主要按钮文案尽量保持统一，避免一个系统里出现过多同义按钮。
