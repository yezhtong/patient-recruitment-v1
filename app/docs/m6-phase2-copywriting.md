# M6 第二阶段 · 四态文案稿

> 每个页面落地时必出的 **loading / empty / error / success** 四态文案统一在这里。
> 原则：患者视角禁用行话；运营视角禁用 DB 术语；**不写医疗承诺**。

---

## 文案通用规则

1. **字段名英文、展示中文**（延续 PRD 通用约束 4）
2. **DB 状态值**（`new`、`pending`、`enrolled` 等）**一律翻译**，禁止裸露在 UI
3. **数字**用阿拉伯数字，不用「五条」「二十条」
4. **Loading**：不用 「加载中…」一刀切，按场景写；短操作（< 500ms）可省略
5. **Empty**：先说事实，再给下一步动作（按钮 / 提示入口）
6. **Error**：先说发生了什么，再说怎么恢复；不暴露技术细节（stack trace / SQL error）
7. **Success**：能做下一步操作的，直接给按钮；不能的，简单 toast 即可

## T1 · KPI 看板（`/admin`）

### Loading
- 服务端渲染首屏，无 loading（Next.js SSR 已经在等数据）
- 时间范围切换（`?range=7d/30d`）是 URL 跳转，浏览器自带 loading，不另做

### Empty
| 区块 | 零态文案 |
|---|---|
| KPI 卡（新线索 / 新报名）| value = 0、hint 改显示「近 7 天暂无数据」（而非「vs 上 7 天 0 条」） |
| KPI 卡（待跟进 / 社区待审 / 招募中 / 累计入组）| value = 0、hint 改显示「暂无」 |
| 线索漏斗 | total=0 时隐藏 4 个空框，显示：「暂无线索数据，等患者提交预筛后展开」 |
| 报名阶段分布 | total=0 时显示：「暂无报名数据」（已有兜底，沿用） |
| Top 5 热门试验 | 查询结果为空时显示：「近 30 日暂无新增线索，请检查招募推广链路」 |
| 待办清单 | 三个待办数都为 0 时显示：「全部处理完毕，休息一下 ☕」（整个区块还是显示的，只是三个数都是 0） |

### Error
- 任意查询失败（Prisma 抛错），该区块显示：「加载失败，请刷新页面；持续失败请联系技术支持」
- **不阻塞整页**：单张卡失败其他卡照常显示

### Success
- 看板是只读页，无 success 态

## T2 · CSV 导出（`/admin/leads` + `/admin/applications`）

### Loading
- 按钮文案在 `pending` 时切换为「导出中…」，按钮 `disabled=true`（已实现，沿用）
- 大于 5 秒未返回时，按钮下方显示小字「数据较多，请稍候…」

### Empty
- 导出结果 0 行时：**不拦截**，仍下载带表头的空 CSV（方便运营拿模板）
- 下载完成后按钮旁短暂提示（3s）：「已下载 0 条，当前筛选无匹配结果」

### Error
| 情境 | 文案 |
|---|---|
| 401 未登录 | 「登录已过期，请重新登录」+ 跳 `/admin/login` |
| 403 权限不足（operator） | 「仅管理员可导出敏感数据」 |
| 500 服务器错误 | 「导出失败，请稍后重试；持续失败请联系技术支持」 |
| 结果被 5 万行上限截断 | Toast：「本次导出前 5 万条，请缩小筛选范围后重新导出」（A1 补丁） |

### Success
- 浏览器默认下载提示即可，不再加 toast（避免打扰）
- 按钮旁可考虑灰色小字「上次导出：14:30」（V2 做，见 backlog V2-4）

## T3 · FAQ 管理（`/admin/faq`）

### Loading
- 列表 SSR，无 loading
- 表单提交时按钮文案「保存中…」、`disabled=true`（已实现）

### Empty
| 场景 | 文案 |
|---|---|
| 库里完全没有 FAQ | 「还没有 FAQ，点击上方「+ 新建 FAQ」开始」（已实现，沿用） |
| 筛选后无结果 | 「当前筛选无匹配 FAQ，请更换条件或清空筛选」+ 「清空筛选」链接 |

### Error
| 场景 | 文案 |
|---|---|
| slug 冲突 | 「slug 已存在，请换一个（可留空由系统生成）」（配合 A2） |
| slug 格式不合法 | 「slug 只允许英文小写、数字、短横，如 `how-to-apply`」 |
| 必填项为空 | 「{字段名}不能为空」（具体字段名） |
| 答案超长 | 「答案不能超过 5000 字，请精简后重试」 |
| 保存服务器错误 | 「保存失败，请稍后重试」 |

### Success
| 场景 | 文案 |
|---|---|
| 新建成功 | 跳回列表；列表顶部 toast「已创建 FAQ『{问题前 20 字}』」（3s 自动消失） |
| 编辑保存成功 | 原地显示绿条「已保存」（已实现） |
| 发布/下架成功 | 按钮即时切换文字（「下架」↔「上架」），无额外 toast |
| 删除成功 | 跳回列表；toast「已删除 FAQ『{问题前 20 字}』，无法恢复」 |

## T4 · 社区分区管理（`/admin/community/groups`）

### Loading
- 列表 SSR；表单保存按钮 pending 状态切换「保存中…」

### Empty
| 场景 | 文案 |
|---|---|
| 库里没有分区（新系统） | 「还没有分区，点击上方「+ 新建分区」开始」（已实现，沿用） |
| 全部分区都已停用 | 「所有分区已停用，患者端 `/community` 会显示『社区维护中』」—— 提示运营至少启用一个 |

### Error
| 场景 | 文案 |
|---|---|
| slug 冲突 | 「slug 已被其他分区使用」 |
| slug 格式错误 | 「slug 只允许英文小写、数字、短横，如 `rare-disease`」 |
| 删除有帖子的分区（A4 改软删后不会再出现，但兜底） | 「该分区已有 {N} 条帖子，建议停用而非删除」 |
| diseaseTag 改动但用户取消 confirm（A5） | 不报错，回到编辑页原状 |
| 服务器错误 | 「操作失败，请稍后重试」 |

### diseaseTag 改动 confirm 文案（A5 专属）
- 弹窗标题：「确认修改病种标签？」
- 正文：「当前有 {N} 条试验按原标签『{oldTag}』关联到本分区，修改后试验详情页的「病友讨论」入口将不再匹配本分区。是否继续？」
- 按钮：「取消」| 「确认修改」
- N=0 时不弹，直接保存

### Success
| 场景 | 文案 |
|---|---|
| 新建成功 | 跳回列表 + toast「已创建分区『{name}』」 |
| 编辑保存成功 | 原地绿条「已保存」 |
| 启用 / 停用 | 开关即时切换，无额外 toast |
| 软删成功（A4 后） | toast「已停用分区『{name}』，患者端不再显示」 |

### Loading

- 批量导入「预览」/「确认导入」按钮 pending 时切换文案 + disabled

### Empty
| 场景 | 文案 |
|---|---|
| 词库完全为空 | 「还没有敏感词，点击上方「批量导入」从词库模板快速导入」（去掉原文的文件路径暴露） |

### Error
| 场景 | 文案 |
|---|---|
| 关键词重复 | 「关键词已存在」 |
| 关键词单字（A7） | 「关键词至少 2 个字符，单字会拦截正常讨论」 |
| 批量导入格式错误 | 「第 {行号} 行格式错误（应为 level\|type\|keyword）」（已实现） |
| 批量导入校验失败 | 「第 {行号} 行校验失败：{具体原因}」（已实现） |
| 服务器错误 | 「操作失败，请稍后重试」 |

### 批量导入两步流程文案（A6 专属）

**第一步：粘贴 → 预览**
- 按钮：「预览导入结果」
- pending 文案：「预览中…」
- 预览表显示：「将新增 {N} / 更新 {M} / 失败 {K} 条」
  - N / M 条行：（折叠，可展开查看前 10 条）
  - K 条行：错误行号 + 原因列表（最多 10 条，超出显示「… 及其他 {总数-10} 条」）
- 按钮区：「重新编辑」（返回粘贴框，不清空内容）| 「确认导入」（主按钮，绿色）

**第二步：确认 → 写库**
- 按钮切换为「确认导入」，pending 文案「导入中…」
- 成功后跳回列表页，顶部 success toast：「批量导入完成：新增 {N} / 更新 {M} / 失败 {K} 条」（已实现，沿用）

### Success
| 场景 | 文案 |
|---|---|
| 新建单条 | 跳回列表 + toast「已添加敏感词『{keyword}』」 |
| 编辑保存 | 原地绿条「已保存」 |
| 启用 / 停用 | 开关即时切换 |
| 删除 | toast「已删除敏感词『{keyword}』」 |
| 批量导入 | 见上方两步流程 |

## 公共 Toast / 对话框组件文案

### 删除二次确认（所有模块共用）
- 标题：「确认删除？」
- 正文：「{对象名}删除后无法恢复（或：停用后可重新启用）」
  - 分区软删（A4 后）：「该分区将被停用，患者端不再显示，但历史数据保留，日后可重新启用」
  - FAQ 删除：「该 FAQ 删除后将从患者端 `/faq` 移除，无法恢复」
  - 敏感词删除：「该敏感词删除后将不再生效，历史命中记录保留」
- 按钮：「取消」| 「确认删除/停用」（危险色）

### 权限不足统一文案
- 文案：「当前账号权限不足，仅管理员可执行此操作。如需操作请联系管理员」
- 不显示具体缺哪个 role（不暴露权限模型细节）

### 会话过期统一文案
- 文案：「登录已过期，请重新登录」+ 自动跳 `/admin/login`
