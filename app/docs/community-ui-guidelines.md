# 九泰临研 V1 · 患者社区模块 UI 设计指引

> 版本：V1 · 2026-04-19 · 用于 M5 实现
>
> 目标：把社区做成**可信、温暖、不廉价**的病友互助空间，延续站内既有 editorial × modern tech 风格（墨绿 + 暖米 + 暖橙 + 电青绿），让中老年患者与家属读得舒服、用得放心。

## 一、全局基调

### 色彩语义（复用 design-system.css token）
| 用途 | 令牌 | 场景 |
|---|---|---|
| 页面背景 | `--cream-50` / `--cream-100` | 社区首页、病种页浅背景 |
| 内容卡片背景 | `--cream-0` (白) | 帖子卡片、评论卡 |
| 强调 / 官方徽标背景 | `--ink-900` | 精选徽章底、官方账号徽标 |
| 主 CTA（立即预筛、发布帖子） | `--accent` (#ff5a1f) | 只留给"强转化"动作 |
| 安全 / 数据提示 | `--lime` / `--lime-soft` | "已通过审核"、"官方精选"等正向徽章 |
| 警告（敏感词命中） | `--warning-500` / `--warning-50` | 待审核条、敏感词拦截提示 |
| 拦截（高风险拒绝） | `--danger-500` / `--danger-50` | 发帖被拒绝的强提示框 |
| 中灰文本（说明、元信息） | `--gray-500` / `--gray-600` | 时间戳、匿名编号、分类标签 |

**情绪规则**：
- 避免"论坛感"——不用高饱和蓝、紫、粉；不用大面积渐变。
- 避免"游戏化"——不用积分、等级、勋章视觉。
- **一个页面只允许一个主 CTA**，发帖 CTA 与预筛 CTA 不得同时用 `--accent` 色占领视线。社区页主 CTA 是"立即预筛"（因为它是 North Star 转化动作），发帖是次级 CTA（墨绿底 `--ink-900`）。

### 字号 & 行距
- 正文最小 **16px**，帖子正文用 17–18px（中老年友好）
- 行距 **1.7** 起（正文 1.75）
- 标题衬线 `Instrument Serif`，列表元信息用 `JetBrains Mono` 小字（11–12px、letter-spacing .08em、uppercase 中文保持原字形）
- 帖子标题 h3 级别：24–28px serif

### 圆角 & 阴影
- 卡片 `--r-xl` (16px) 或 `--r-2xl` (20px)，避免过圆（那是消费 app）
- 徽章 / 标签：`--r-pill`
- 阴影一律用细边框代替（`border: 1px solid var(--ink-100)`），编辑感更强、比阴影更"干净"
- 精选帖：左边 4px 纯色竖线（`--accent`），不要加阴影

### 间距节奏
- 卡片内 padding：`24px 28px`（桌面）/ `18px 20px`（移动）
- 卡片垂直间距：16px
- 区块之间：48px
- 8px 为最小单位（design-system 已统一）

## 二、通用组件约定

### 1. 帖子卡片 · 3 种视觉层级

所有帖子都是"标题 + 摘要 + 元信息"的卡片，通过**结构和边框**区分层级，不靠大色块。

#### A. 普通帖（默认）
```
┌─────────────────────────────────────────────────┐
│ [提问]  [难治性高血压]                           │  ← 标签行，pill，灰边
│                                                 │
│ 做完 RDN 手术第 3 周，还需要继续吃降压药吗？      │  ← h3, serif, 22px
│ 医生说血压还没稳定。我老伴儿是上周安贞做的…       │  ← 摘要 2 行，ink-700，截断
│                                                 │
│ 匿名患者 #A3F2 · 2 小时前 · 3 条回复             │  ← mono 11px, gray-500
└─────────────────────────────────────────────────┘
```
- 背景 `--cream-0`
- `border: 1px solid var(--ink-100)`
- hover：`border-color: var(--ink-300)`，**不位移**
- 点击整卡跳转，鼠标 cursor pointer

#### B. 精选帖
- 在普通帖基础上，**左边 4px 纯色竖线** `--accent`
- 左上角加一个小徽章：`★ 精选` · 黑底白字 pill · mono 11px uppercase
- 标题字号放大到 26–28px
- 摘要可以展开 3 行而不是 2 行

#### C. 官方 / 运营回答
- 背景换为 `--ink-900`（墨绿深底），文字 `--cream-50`
- 左上徽章：`✓ 官方答复` · `--lime` 底 + `--ink-900` 字 · pill
- 标题字号 24px，`color: var(--cream-50)`
- 摘要文字 `color: rgba(253,250,243,.8)`
- 元信息行用 `color: var(--lime)` 的 mono 小字
- 作用：在病种社区顶部或帖子详情"相关答疑"位承载"官方答复"型内容

### 2. 标签组（category / disease / post_type）
- pill 形态，padding `4px 12px`，`--r-pill`
- 字体 `Inter 12px / medium`
- 颜色：
  - 病种标签（如"难治性高血压"）：`--ink-100` 底 + `--ink-700` 字
  - 帖子类型"提问"：`--warning-50` 底 + `--warning-700` 字
  - 帖子类型"经验分享"：`--lime-soft` 底 + `--ink-700` 字
- 一个卡片/详情页**最多展示 2 个标签**，其他标签折叠到详情页底部"相关"区

### 3. 作者信息行
- 实名：`头像圆 28px + 昵称 + 时间`
- 匿名：`灰色圆（无头像，用字母 A）+ 匿名患者 #<4 位大写十六进制> + 时间`
  - #4 位编号取自 userId 的 hash 后 4 位，同一人在同一病种社区里编号稳定（可以看到"匿名患者 #A3F2 又发了一条"，但跨病种不可关联）
- 时间：相对时间 "刚刚 / 2 小时前 / 昨天 / 3 天前 / 2026-04-10"
- 官方账号：头像改为 `--lime` 圆 + `JT` mono 字，昵称后加 `✓ 官方` pill

### 4. CTA 按钮
- **主 CTA（立即预筛）**：`--accent` 底 + 白字 + 右侧箭头 `→`，用在"帖子详情页嵌入的相关试验卡"和"病种页相关试验推荐"
- **次 CTA（发布帖子）**：`--ink-900` 底 + `--cream-50` 字
- **轻量动作（回复、收藏、举报）**：文字按钮 + 下划线 hover，不用边框按钮

### 5. 免责声明
必须在以下位置至少出现一次：
- `/community` 首页头部 hero 下方
- 发帖页表单顶部
- 每个帖子详情页在正文之后、评论区之前

文案统一：
> ★ 提醒：社区内容仅供病友交流参考，**不能替代医生的专业建议**。遇到紧急不适请直接就医或拨打 120。

样式：
- 背景 `--warning-50` / 边框 `--warning-500` / 字色 `--warning-700`
- 圆角 `--r-md`
- mono 11px 前缀 "★ 提醒"，后接正文 Inter 14px

#### 关键约定
- 正文容器最大宽度 **720px**，保证长文阅读（线长 ~65 字符）
- 正文段落 `<p>` 之间 `margin: 1em 0`（不要用空行）
- 已登录且作者是本人时，头部右上角显示"删除帖子"文字按钮（灰色下划线，不做主按钮）

- `headline` = 帖子标题
- `author.name` = 实名或"匿名患者 #xxxx"
- `datePublished` = createdAt
- `articleBody` = 正文
- `commentCount` = 评论数

### 页面 4：`/community/new` 发帖页

**目标**：让发帖行为"仪式感"—— 让患者在写之前先看到规则，而不是写完才被拒绝。

```
[SiteHeader]

│  发布新帖 / 分享经验                                   │

┌─ 发帖表单（form-card，padding 40 44）────────────────────┐
│                                                       │

│  │ ○──●  匿名发布                           │         │
│  │ 开启后，其他用户会看到 "匿名患者 #A3F2"，     │         │

│  [  发布  →  ]                                         │
└──────────────────────────────────────────────────────┘

```

#### 交互态
1. **输入字数统计**（标题 / 正文）：实时，接近上限时变 `--warning-500`
2. **敏感词拦截**（服务端校验）：
   - 中风险（如"偏方"）→ 提示"该内容需要先人工审核，发布后不会立即可见"；允许提交
   - 高风险（如"微信：139…"）→ **阻止提交**，弹出错误框：
     ```
     ⛔ 这条内容包含不允许发布的元素：
        · 疑似暴露联系方式（手机号）
        · 建议修改后重新发布
     [知道了]
     ```
     错误框用 `--danger-50` 底 + `--danger-500` 边 + `--danger-700` 字，关闭后光标聚焦到首个命中的位置（首命中字符加 `text-decoration-line: underline wavy; text-decoration-color: var(--danger-500);`，保留 5 秒）
3. **提交后**：
   - 低风险 → 直接跳 `/community/posts/[id]`
   - 中风险 → 跳 `/community/submitted?status=pending`，"帖子已提交，等待审核。预计 8 小时内审核完成。"
4. **未登录访问 `/community/new`** → 跳 `/auth?next=/community/new`

### 页面 5：`/admin/community/*` 审核后台

#### 5a. `/admin/community/posts` 帖子审核列表
- 复用 admin.css 的 `.admin-table`、`.admin-filters`、`.chip`
- 筛选：审核状态（pending / approved / rejected / hidden / featured）、病种、是否匿名、风险命中
- 列：时间 / 摘要（标题截断 + 正文前 40 字）/ 作者手机号脱敏 / 是否匿名 / 病种 / 状态 chip / 风险命中数
- 每行右侧 4 个快捷动作：`通过` / `驳回` / `隐藏` / `精选`
  - 驳回和隐藏需要二次确认
  - 通过时自动 revalidate 前台病种社区
- 风险命中单独一列，显示命中词数（`🔔 2 项`，点击弹出抽屉显示具体命中位置）

#### 5b. `/admin/community/posts/[id]` 单条帖子审核
- 左侧：原帖完整展示（模拟前台样式但带"仅后台可见"水印）
- 右侧 sticky：
  - 动作按钮（通过 / 驳回 + 原因输入 / 隐藏 / 精选）
  - 风险命中面板（所有敏感词、所在字段、原文片段）
  - 作者账号（不脱敏、可跳转 `/admin/applications?q=<phone>`）
  - 审核日志（操作人 + 时间 + 动作 + 原因）

#### 5c. `/admin/community/groups` 社区分区配置
- 表格：分区名 / 病种标签 / 启用开关 / 排序 / 帖子数 / 本周新增
- 详情编辑：分区简介（运营文案）、推荐试验（多选试验 id）、推荐 FAQ（多选 faq id）

#### 5d. `/admin/community/sensitive-hits` 风险命中
- 按词统计：`"包入组"` 命中 34 条、`"偏方"` 命中 12 条
- 点击进入命中列表，每条可直接跳到原帖审核页

## 四、匿名发帖 UX 细节

1. **匿名 ID 稳定性**：
   - 每个用户在同一个 `groupId` 内，匿名编号是 `sha1(userId + groupId).slice(-4).toUpperCase()`
   - 跨分区编号不同，避免跨病种追踪
   - 运营后台始终能看到真实 userId 和手机号
2. **Toggle 设计**：
   - 用原生 `<input type="checkbox">` + CSS 伪类做滑块，不引组件库
   - 默认**关闭**（实名），切换需要主动动作（保守策略）
   - Toggle 下方固定说明文本（Inter 13px / gray-500）
3. **"删除自己帖子"**：
   - 匿名帖删除时只消除前台，后台保留审计记录
   - 删除前二次确认："删除后无法恢复，但运营仍保留一份备份用于审计。"

## 五、交互与动效

- 所有过渡 **150–200ms ease-out**，不做弹簧效果（医疗场景"稳重"优先）
- 卡片 hover 只改边框色，**不做 translate 或 scale**（防止中老年用户困惑）
- 评论提交、状态流转后用 `<div aria-live="polite">` 输出反馈文字（屏幕阅读器友好）
- 加载骨架：帖子列表用灰色占位块（`--ink-100` 底 + 14px 圆角）
- 错误提示统一用 `role="alert"` + `--danger-50` 底 + `--danger-500` 边

---

## 六、无障碍基线（必过）

- 正文最小 16px，详情页正文 17–18px
- 文字与背景对比度 ≥ 4.5:1（墨绿 `--ink-900` vs 暖米 `--cream-50` 对比足够）
- 所有 icon-only 按钮必须有 `aria-label`
- 敏感词高亮采用 `text-decoration-line: underline wavy` 而非纯色，颜色盲可识别
- 匿名 toggle 是原生 checkbox，Tab 可达
- 评论区在屏幕阅读器里读作 `list` + `listitem`

## 七、数据模型对应（给 backend 的锚点）

| UI 元素 | 对应模型字段 |
|---|---|
| 帖子标签（病种） | `CommunityGroup.diseaseTag` |
| 帖子类型 Pill | `CommunityPost.postType`（question / experience） |
| 匿名编号 #A3F2 | 渲染时计算：`sha1(userId + groupId).slice(-4)` |
| 精选徽章 | `CommunityPost.isFeatured` |
| 官方徽章 | `CommunityPost.authorRole = 'operator'` |
| 风险命中 | `CommunitySensitiveHit[]`（关联到 post 或 comment） |
| 审核状态 | `CommunityPost.reviewStatus`（pending / approved / rejected / hidden / featured） |
| 审核日志 | `CommunityModerationLog` |

## 八、切忌

- ❌ 不用表情包做 icon（站内已有规则）
- ❌ 不做点赞、积分、等级、收藏夹（V1 Out of Scope）
- ❌ 不做站内私信（V1 Out of Scope，引导到预筛或客服）
- ❌ 不在帖子卡片上堆 > 3 个可点击元素（中老年用户易误触）
- ❌ 不把"发布新帖"用 `--accent` 色（保留给预筛转化）
- ❌ 不在敏感词拦截时沉默丢弃，必须明确说出哪条规则被触发
