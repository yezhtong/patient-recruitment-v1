## 〇、核心决策摘要（给工程师 2 分钟读懂）

| 维度 | 决策 | 理由 |
| --- | --- | --- |
| **hero 动效** | **crossfade（淡入淡出 + 轻微 1.02 → 1.00 scale）** | 中老年用户，slide 横向位移在文字 overlay 上视觉噪点大；crossfade 最稳 |
| **实现方案** | **JS 控制状态 + CSS 动画类切换** | scroll-snap 可实现但无法稳定控制 "当前活动索引"、暂停、读屏器 ARIA 状态；overlay 与图同步切换更是需要状态同步 |
| **自动间隔** | **6000ms**（中老年友好，官方建议范围 5-7s 取中位） | |
| **overlay 与图** | **同步 crossfade**（不做先淡出后淡入） | 保证 label+text 对得上当前图，避免错位被误解 |
| **暂停机制** | hover 暂停 + focus-within 暂停 + `prefers-reduced-motion` 下禁动画改点按切换 | |
| **指示器** | 下方居中 dot 组，当前 dot 拉长为胶囊 | 比数字含蓄，比箭头省空间 |
| **箭头** | 桌面 hover 显示、移动端常驻（tap 区 44×44） | 移动端无 hover 态，箭头必须可见 |
| **token 新增** | **0**（全复用 design-system.css 既有 token） | |

## 一、首页 hero 轮播视觉规格

### 1.1 容器尺寸

右列图片容器沿用 `.hero-photo` 的 `aspect-ratio: 4 / 5`，轮播外壳（新增 `.hero-carousel`）继承此比例。

| 断点 | 容器行为 | 高度参考 |
| --- | --- | --- |
| ≥ 1280 | 右列约占 `1fr`（`.hero__grid` 已定 `1.15fr 1fr`），比例 4:5 | ~ 620px |
| 960 – 1279 | 同上，略窄 | ~ 520px |
| 641 – 959 | 单列，图下移到文案下方，宽度 100%（container 内），比例改为 **4:3** 减少滑动距离 | ~ 360px |
| ≤ 640 | 宽度 = `calc(100vw - var(--container-pad) * 2)`，比例 **4:5** 恢复（手机竖屏更自然） | ~ 440px |
| ≤ 400 | 同上，图不变，overlay 字号按 §1.3 降级 | ~ 420px |

### 1.2 overlay 位置 + 安全区

- 位置：贴底、满宽（延续现有 `.hero-photo__overlay` 的 `left:0 right:0 bottom:0`）
- 内边距：桌面 `28px 28px 24px`、≤ 640 `20px 20px 18px`、≤ 400 `16px 16px 14px`
- **图片安全区**：运营上传图时需预留 **图底部 25% 高度不能是关键主体**（人脸、医生仪器、文字）——此约束在后台上传页用说明文案 + 虚线辅助框告知（见 §三）
- overlay 背景已经是 `linear-gradient(to top, rgba(10,20,17,.78) 0%, rgba(10,20,17,.35) 60%, transparent 100%)`，**不改**

### 1.3 overlay 字号对比度

| 元素 | 字号（桌面 / ≤640 / ≤400） | token | 对比度（对 `rgba(10,20,17,.78)` 底色，≈ #0A1411） |
| --- | --- | --- | --- |
| label（英文 mono） | 11 / 11 / 11 | `var(--lime)` `#c8ff4a` | **15.2:1** ✓ |
| text（中文 serif） | 20 / 17 / 16 | `var(--cream-50)` `#fdfaf3` | **17.1:1** ✓ |

两项均远高于 WCAG AA 4.5:1 门槛，即使 overlay 底色降到 55% 透明度仍过线。

### 1.4 切换动效规格（crossfade 方案）

```css
/* 所有 slide 叠在同一容器内 position:absolute 铺满 */
.hero-carousel__slide {
  position: absolute; inset: 0;
  opacity: 0;
  transform: scale(1.02);
  transition:
    opacity 480ms var(--ease-in-out),
    transform 600ms var(--ease-out);
  pointer-events: none;
}
.hero-carousel__slide[data-active="true"] {
  opacity: 1;
  transform: scale(1);
  pointer-events: auto;
}
```

- **时长**：不透明度 480ms、缩放 600ms——感知上略错峰，落地更柔和
- **easing**：
  - opacity 用 `var(--ease-in-out)`（`cubic-bezier(.65,0,.35,1)`）——对称
  - transform 用 `var(--ease-out)`（`cubic-bezier(.16,1,.3,1)`）——开始快结束稳
- overlay 挂在 slide 内部，**随同一次 opacity 变化同时切换**（不做双阶段）

### 1.6 前后箭头

| 属性 | 值 |
| --- | --- |
| 尺寸 | 外层 44×44，内部 SVG 20×20 |
| 位置 | 左右两侧垂直居中，距边 12px |
| 背景 | `background: rgba(253,250,243,.88)`；`backdrop-filter: blur(6px)` |

### 1.7 自动播放与暂停

- 间隔：**6000ms**
- 暂停触发器（任一命中即暂停）：
  1. `:hover`（鼠标悬停容器内任意处）
  2. `:focus-within`（键盘焦点在容器内）
  3. 用户主动点击指示器/箭头后，暂停 **10s** 再恢复（避免用户刚切到想看的图又被自动切走）
  4. `prefers-reduced-motion: reduce` → 永久暂停，首屏停第 1 张
  5. 页面 `document.visibilityState === "hidden"` → 暂停（省电/守规）
- **不提供显式"暂停/播放"按钮**——中老年用户看到会疑惑；隐性暂停已覆盖所有场景

### 1.8 单张（仅 1 条数据）降级

- 不渲染指示器、不渲染箭头、不启动自动播放
- 样式与 M8.3 之前完全一致，无视觉回退

## 四、a11y 专项

### 4.1 轮播语义

```html
<section
  class="hero-carousel"
  role="region"
  aria-roledescription="轮播图"
  aria-label="首页主图"
>
  <div class="hero-carousel__viewport" aria-live="polite" aria-atomic="false">
    <div class="hero-carousel__slide" data-active="true" aria-hidden="false">...</div>
    <div class="hero-carousel__slide" data-active="false" aria-hidden="true">...</div>
  </div>
  <div class="hero-carousel__dots" role="tablist" aria-label="选择轮播图">
    <button role="tab" aria-selected="true" aria-label="第 1 张，共 3 张">...</button>
  </div>
  <button class="hero-carousel__arrow hero-carousel__arrow--prev" aria-label="上一张">...</button>
  <button class="hero-carousel__arrow hero-carousel__arrow--next" aria-label="下一张">...</button>
</section>
```

- `aria-live="polite"` 放在 viewport 上——切换时读屏器会播报新 slide 的 alt + overlay 文字
- 非活动 slide 必须 `aria-hidden="true"` 且内部可交互元素 `tabindex="-1"`

### 4.2 prefers-reduced-motion

```css
@media (prefers-reduced-motion: reduce) {
  .hero-carousel__slide { transition: none; transform: none; }
  .hero-carousel__dots button { transition: none; }
}
```

+ JS 侧：读到该 media query 为 true 时，**禁用 autoplay**，只响应用户主动点击；图切换时直接切 `data-active`，不带 fade

### 4.3 键盘操作

| 按键 | 行为 | 触发条件 |
| --- | --- | --- |
| Tab | 按序进入 prev 箭头 → 每个 dot（tablist 内一次 tab，方向键切换）→ next 箭头 → slide 内链接 | 始终 |
| ←/→ | 上一张/下一张 | 焦点在 tablist 或容器内 |
| Home/End | 第一张/最后一张 | 焦点在 tablist |
| Space/Enter | 切换到当前聚焦 dot 对应的 slide | dot 聚焦 |

### 4.4 焦点可见环

全站已有 `:focus-visible { box-shadow: var(--shadow-focus) }`（暖橙外环），**不改**。轮播内按钮继承即可。

## 七、实现边界建议

### 7.1 CSS-only vs JS

**推荐 JS 状态驱动 + CSS 过渡**。理由：

| 维度 | CSS-only (scroll-snap) | JS 方案 | 结论 |
| --- | --- | --- | --- |
| crossfade | 难（scroll-snap 本质是 slide） | 原生支持 | JS 赢 |
| 自动播放 | 做不到（CSS 无 timer） | `setInterval` | JS 赢 |
| 暂停/恢复/用户交互后延迟 | 做不到 | 直接控 | JS 赢 |
| 键盘/读屏器 `aria-live` 切换 | 很难 | 直接控 | JS 赢 |
| overlay 与图同步切换 | 可做但需 IntersectionObserver | 天然同步 | JS 赢 |
| 包体积 | 0 KB | ≤ 2 KB | 可接受 |

实现用 React `useState` + `useEffect` + `useRef`，**不引入任何第三方轮播库**（swiper/embla/keen-slider 全部禁用）。

### 7.2 overlay 出入场

**同步** crossfade（overlay 是 slide 的一个 DOM 子节点，跟 slide 一起 opacity 过渡）。

**不做**「先淡出 overlay → 切图 → 淡入新 overlay」的双阶段方案——会拉长总时长到 > 1s，中老年用户还没反应过来就又切走了。

### 7.3 图片加载

- 第 1 张：`loading="eager"`，`fetchpriority="high"`
- 第 2～n 张：`loading="lazy"`，但在 autoplay 启动前由 JS 预加载（`new Image().src = url`）——避免第一次切换时白屏
