## 🚀 服务器部署步骤（复制即可执行）

### 前提
- 服务器已装 Docker 与 docker compose：
  ```bash
  docker --version && docker compose version
  ```
- 把上面 3 个文件一起上传到服务器同一个目录，例如 `/opt/patient-recruitment/`

### 第 1 步 解压代码
```bash
cd /opt/patient-recruitment
unzip 患者招募-app.zip -d app
cd app
```

### 第 2 步 生成配置文件（只做一次）
```bash
cp .env.example .env

# 生成随机 Session 口令并自动写入 .env
SP=$(openssl rand -base64 48)
sed -i "s|SESSION_PASSWORD=.*|SESSION_PASSWORD=\"$SP\"|" .env

# 把站点地址改成你的服务器公网 IP 或域名
# 用编辑器打开 .env，修改 NEXT_PUBLIC_SITE_URL 这一行
nano .env
```

### 第 3 步 构建并启动容器
```bash
docker compose up -d --build
```
首次构建约 3–5 分钟（国内镜像源已配置，走 npmmirror）。

### 第 4 步 导入初始数据库（首次必做）
```bash
# 把上传的初始库拷进容器挂载的数据目录
docker compose cp ../初始数据_dev.db app:/app/data/prod.db
docker compose restart
```

### 第 5 步 验证
浏览器访问：`http://服务器IP:3000`

应看到首页、试验列表、社区、FAQ 正常渲染。
默认账号：
- 管理后台：`/admin/login` · admin / admin123
- 患者登录：`/auth` · 任意手机号 + 验证码 `123456`（dev 固定码，商用接入短信前保持）

## 🔧 常用运维命令

| 操作 | 命令 |
|------|------|
| 查看实时日志 | `docker compose logs -f app` |
| 重启服务 | `docker compose restart` |
| 停止服务 | `docker compose down` |
| 更新代码后重建 | `docker compose up -d --build` |
| 备份数据库 | `docker compose cp app:/app/data/prod.db ./backup-$(date +%Y%m%d).db` |
| 进入容器排查 | `docker compose exec app sh` |

## ⚠️ 注意事项

1. **SESSION_PASSWORD 必须改**，否则 iron-session 在生产会拒绝启动；第 2 步已自动写入随机值。
2. `.env` 文件含密钥，**请勿上传到任何代码仓库或网盘**。
3. 数据库是 SQLite 单文件，**不支持多实例横向扩展**；用户量超过 1 万再迁 MySQL（项目已内置 `scripts/switch-db.ts` 切换脚本）。
4. 端口默认 3000，如要换端口或加 HTTPS，修改 `docker-compose.yml` 的 ports，并建议前面挂一层 Nginx 反向代理。
5. **每月至少一次**执行备份命令，把 `backup-*.db` 下载到本机或 NAS。

## ❓ 出问题时先看

- `docker compose logs -f app` 看报错
- 容器一直 restart：通常是 `.env` 的 `SESSION_PASSWORD` 太短或 `DATABASE_URL` 改错了
- 首页 500：进容器 `ls -la /app/data/` 确认 `prod.db` 存在
- 端口不通：服务器防火墙/安全组需放行 3000

有问题把 `docker compose logs --tail=200 app` 的输出发给研发。
